# Git 仓库设置指南

## ⚠️ 重要：首次使用必须先配置用户信息

在提交代码之前，必须先配置Git用户信息：

```bash
# 方式1：仅在此项目中使用（推荐）
git config user.name "Your Name"
git config user.email "your.email@example.com"

# 方式2：全局配置（所有项目使用）
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

**示例：**
```bash
git config user.name "张三"
git config user.email "zhangsan@example.com"
```

配置完成后，可以继续下面的步骤。

## 2. 查看当前状态

```bash
git status
```

## 3. 添加文件到暂存区

```bash
# 添加所有文件
git add .

# 或者只添加特定文件
git add TestTool/src/
git add TestTool/Seq/
```

## 4. 提交更改

```bash
git commit -m "Initial commit: TestTool v0.4"
```

## 5. 创建远程仓库（GitHub/GitLab/Gitee等）

1. 在 GitHub/GitLab/Gitee 上创建新仓库
2. 获取仓库地址（例如：https://github.com/username/TestTool.git）

## 6. 添加远程仓库

```bash
git remote add origin https://github.com/username/TestTool.git
```

## 7. 推送到远程仓库

```bash
# 首次推送
git push -u origin main

# 或者如果默认分支是 master
git push -u origin master
```

## 8. 后续操作

### 查看更改
```bash
git status
git diff
```

### 提交更改
```bash
git add .
git commit -m "描述你的更改"
git push
```

### 查看提交历史
```bash
git log
git log --oneline
```

### 创建分支
```bash
git checkout -b feature/new-feature
git push -u origin feature/new-feature
```

## 注意事项

- `.gitignore` 文件已配置，会自动排除日志文件、CSV数据文件、__pycache__等
- 敏感配置文件（如包含密码的config.yaml）建议不要提交，或使用加密存储
- 定期提交代码，保持提交信息清晰明确
