## 📝 使用方法

### 基本用法

```yaml
steps:
  # 步骤1: 扫描序列号
  - id: step_1
    type: scan.sn

  # 步骤2: 写入PN（使用扫描的SN）
  - id: step_2
    type: engineer.test
    params:
      command: 'pn=1,s100,${sn}%'  # ${sn} 自动替换
      robot_ip: '192.168.125.2'
```

### 执行过程

```
1. 用户扫描 SN: VITA2024001
   ↓
2. SN 保存到上下文: ctx.set_sn("VITA2024001")
   ↓
3. step_2 读取命令: pn=1,s100,${sn}%
   ↓
4. 替换变量: ${sn} → VITA2024001
   ↓
5. 执行命令: pn=1,s100,VITA2024001%
```

## 📖 PN 命令说明

### 命令格式
```
pn=<op>,<device>,<value>%
```

### 参数详解

| 参数 | 说明 | 可选值 |
|------|------|--------|
| `op` | 操作类型 | `0`=读取, `1`=写入 |
| `device` | 设备类型 | `sn`=主板, `s100`=S100模块, `x5`=X5模块 |
| `value` | 序列号 | 任意字符串（写入时需要） |

### 常用命令

```yaml
# 读取命令
command: 'pn=0,sn%'        # 读取主板SN
command: 'pn=0,s100%'      # 读取S100 PN
command: 'pn=0,x5%'        # 读取X5 PN

# 写入命令（使用变量）
command: 'pn=1,sn,${sn}%'     # 写入主板SN
command: 'pn=1,s100,${sn}%'   # 写入S100 PN  ← 当前使用
command: 'pn=1,x5,${sn}%'     # 写入X5 PN
```

## ⚠️ 注意事项

### 1. 执行顺序

**必须先扫描 SN，再使用 `${sn}`**

✅ 正确:
```yaml
steps:
  - scan.sn          # 先扫描
  - engineer.test    # 再使用 ${sn}
```

❌ 错误:
```yaml
steps:
  - engineer.test    # 使用 ${sn} 但还未扫描！
  - scan.sn
```

---

### 2. 变量格式

- ✅ 正确: `${sn}`
- ❌ 错误: `$sn`, `{sn}`, `$(sn)`

---

### 3. 机器人连接

ping 192.168.125.2
