# 生产线测试工具架构分析

## 系统概述

这是一个基于Python和PySide6开发的智能硬件生产线测试工具，采用模块化架构设计，支持双端口并行测试、多厂商MES集成、权限管理、系统自检等功能。

## 整体架构

### 1. 分层架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                    用户界面层 (UI Layer)                      │
├─────────────────────────────────────────────────────────────┤
│  MainWindow │ PortPanel │ ConfigDialog │ SequenceEditor     │
└─────────────────────────────────────────────────────────────┘
                              │
┌─────────────────────────────────────────────────────────────┐
│                   应用服务层 (App Layer)                      │
├─────────────────────────────────────────────────────────────┤
│  Worker │ I18n │ LogBridge │ SequenceModel │ ToolLauncher   │
└─────────────────────────────────────────────────────────────┘
                              │
┌─────────────────────────────────────────────────────────────┐
│                   核心业务层 (Core Layer)                     │
├─────────────────────────────────────────────────────────────┤
│  EventBus │ Scheduler │ Lifecycle │ Messages │ Health       │
└─────────────────────────────────────────────────────────────┘
                              │
┌─────────────────────────────────────────────────────────────┐
│                   功能模块层 (Module Layer)                   │
├─────────────────────────────────────────────────────────────┤
│ TestCases │ MES │ Security │ SelfCheck │ Config │ Logging   │
└─────────────────────────────────────────────────────────────┘
                              │
┌─────────────────────────────────────────────────────────────┐
│                   驱动适配层 (Driver Layer)                   │
├─────────────────────────────────────────────────────────────┤
│  Comm │ Instruments │ UUT │ Fixture │ Protocols            │
└─────────────────────────────────────────────────────────────┘
```

### 2. 核心组件

#### 2.1 事件总线 (EventBus)
- **位置**: `src/core/bus.py`
- **功能**: 提供发布-订阅模式的事件通信
- **特性**: 
  - 支持同步和异步事件处理
  - 支持通配符模式匹配
  - 线程安全的事件分发
  - 错误隔离，单个处理器失败不影响其他处理器

#### 2.2 调度器 (Scheduler)
- **位置**: `src/core/scheduler.py`
- **功能**: 管理定时任务和周期性任务
- **特性**:
  - 支持一次性任务和周期性任务
  - 支持任务重试策略
  - 异步任务执行

#### 2.3 生命周期管理 (LifecycleManager)
- **位置**: `src/core/lifecycle.py`
- **功能**: 管理服务的启动、停止和配置变更
- **特性**:
  - 依赖感知的服务启动顺序
  - 拓扑排序确保依赖关系
  - 配置变更广播

## 模块详细分析

### 1. 人机交互模块 (app/)

#### 1.1 主窗口 (MainWindow)
- **文件**: `src/app/views/main_window.py`
- **功能**: 
  - 双端口并行测试界面
  - 工具栏和菜单管理
  - 测试序列树显示
  - 实时状态监控
- **关键特性**:
  - 支持断点调试
  - 多语言切换
  - 权限控制集成

#### 1.2 端口面板 (PortPanel)
- **文件**: `src/app/views/port_panel.py`
- **功能**:
  - 单个端口的测试状态显示
  - 测试进度和结果展示
  - 实时数据可视化
  - 操作控制按钮

#### 1.3 后台工作器 (PortWorker)
- **文件**: `src/app/worker.py`
- **功能**:
  - 在独立线程中执行测试
  - 发送状态和进度信号
  - 支持暂停、恢复、停止操作
  - 集成新架构的测试执行引擎

### 2. 测试用例模块 (testcases/)

#### 2.1 测试执行引擎 (TestRunner)
- **文件**: `src/testcases/runner.py`
- **功能**:
  - 管理测试序列的执行
  - 步骤结果收集和验证
  - 错误处理和重试机制
  - 测试上下文管理

#### 2.2 测试步骤框架 (IStep)
- **文件**: `src/testcases/step.py`
- **功能**:
  - 定义测试步骤的标准接口
  - 支持步骤的准备、执行、验证、清理
  - 步骤结果数据模型

#### 2.3 步骤注册系统 (Registry)
- **文件**: `src/testcases/registry.py`
- **功能**:
  - 动态注册和管理测试步骤
  - 支持插件式步骤扩展
  - 步骤类型检查和验证

#### 2.4 内置步骤库 (steps/)
- **通信步骤**: 串口/TCP通信
- **仪器步骤**: 电源、万用表等仪器控制
- **UUT步骤**: 被测单元通信和控制
- **MES步骤**: MES系统集成
- **工具步骤**: 延时、条件判断等

### 3. MES集成模块 (mes/)

#### 3.1 MES客户端 (MESClient)
- **文件**: `src/mes/client.py`
- **功能**:
  - 统一的MES接口抽象
  - 工单信息获取
  - 测试结果上传
  - 心跳监控

#### 3.2 适配器模式 (Adapters)
- **文件**: `src/mes/adapters/`
- **功能**:
  - 支持不同MES厂商的适配
  - 协议转换和数据映射
  - 错误处理和重试机制

#### 3.3 工厂模式 (Factory)
- **文件**: `src/mes/factory.py`
- **功能**:
  - 根据配置创建MES客户端
  - 适配器注册和管理
  - 统一的对象创建接口

### 4. 权限管理模块 (security/)

#### 4.1 认证服务 (AuthService)
- **文件**: `src/security/auth_service.py`
- **功能**:
  - 用户登录和会话管理
  - 密码加密和验证
  - 会话超时处理

#### 4.2 权限控制 (RBACService)
- **文件**: `src/security/rbac_service.py`
- **功能**:
  - 基于角色的访问控制
  - 权限检查和验证
  - 角色管理

#### 4.3 审计服务 (AuditService)
- **文件**: `src/security/audit_service.py`
- **功能**:
  - 操作审计日志记录
  - 审计报告生成
  - 安全事件监控

### 5. 系统自检模块 (selfcheck/)

#### 5.1 分阶段检查
- **系统启动检查**: 软件环境、硬件资源、日志系统
- **配置完成检查**: 通信接口、仪器连接
- **测试就绪检查**: 配置文件、测试序列

#### 5.2 检查器系统
- **软件环境检查器**: Python版本、依赖包、环境变量
- **硬件资源检查器**: 磁盘空间、内存、网络
- **通信接口检查器**: 串口、TCP连接、端口冲突
- **仪器连接检查器**: VISA接口、SCPI命令、仪器自检

### 6. 配置管理模块 (config/)

#### 6.1 配置模型 (Pydantic)
- **文件**: `src/config/models.py`
- **功能**:
  - 类型安全的配置定义
  - 自动验证和默认值
  - 配置文档生成

#### 6.2 配置服务 (ConfigService)
- **文件**: `src/config/service.py`
- **功能**:
  - YAML配置文件加载和保存
  - 配置变更通知
  - 敏感信息加密存储

### 7. 日志模块 (app_logging/)

#### 7.1 日志管理器 (LoggingManager)
- **文件**: `src/app_logging/manager.py`
- **功能**:
  - 统一日志管理
  - 多端口日志分离
  - 日志轮转和清理

#### 7.2 分类日志
- **测试日志**: 测试过程和结果记录
- **错误日志**: 错误和异常记录
- **系统日志**: 系统操作和配置变更记录

### 8. 驱动模块 (drivers/)

#### 8.1 通信驱动 (comm/)
- **串口驱动**: 基于pyserial的串口通信
- **TCP驱动**: 基于socket的TCP通信
- **工厂模式**: 统一创建不同类型的传输对象

#### 8.2 仪器驱动 (instruments/)
- **VISA会话管理**: 基于pyvisa的仪器控制
- **SCPI命令封装**: 标准SCPI命令接口
- **电源控制**: 程控电源的完整控制接口

### 9. UUT模块 (uut/)

#### 9.1 UUT适配器 (UUTAdapter)
- **文件**: `src/uut/adapter.py`
- **功能**:
  - 统一的UUT通信接口
  - 多协议支持
  - 连接状态管理

#### 9.2 协议解析 (Protocols)
- **文件**: `src/uut/protocols.py`
- **功能**:
  - 多种通信协议解析
  - 消息格式转换
  - 数据验证和错误检测

## 数据流向分析

### 1. 测试执行数据流

```
用户操作 → MainWindow → PortWorker → TestRunner → IStep → 驱动层 → 硬件设备
    ↓           ↓           ↓           ↓         ↓        ↓
UI更新 ← 信号发送 ← 结果收集 ← 步骤执行 ← 数据返回 ← 响应处理
```

### 2. 配置管理数据流

```
配置文件 → ConfigService → Pydantic模型 → 配置验证 → 服务分发
    ↓           ↓              ↓           ↓         ↓
配置变更 → 事件发布 → EventBus → 服务通知 → 配置更新
```

### 3. MES集成数据流

```
测试结果 → MESClient → 适配器 → MES系统 → 响应处理 → 状态更新
    ↓         ↓         ↓        ↓         ↓         ↓
结果上传 → 协议转换 → 数据映射 → 网络传输 → 结果确认 → UI更新
```

### 4. 日志记录数据流

```
操作/事件 → 日志记录器 → 格式化器 → 处理器 → 文件/远程 → 日志存储
    ↓          ↓           ↓         ↓        ↓         ↓
分类处理 → 级别过滤 → 格式转换 → 输出处理 → 存储管理 → 轮转清理
```

### 5. 权限控制数据流

```
用户操作 → 权限检查 → 角色验证 → 操作执行 → 审计记录 → 日志存储
    ↓         ↓         ↓         ↓         ↓         ↓
登录验证 → 会话管理 → 权限判断 → 功能访问 → 操作追踪 → 安全监控
```

## 关键技术特性

### 1. 异步编程
- 使用asyncio进行异步操作
- 支持并发测试执行
- 非阻塞UI响应

### 2. 事件驱动架构
- EventBus提供松耦合的模块通信
- 支持发布-订阅模式
- 异步事件处理

### 3. 依赖注入
- 通过构造函数注入依赖
- 支持服务生命周期管理
- 便于单元测试

### 4. 插件化设计
- 测试步骤可动态注册
- MES适配器可扩展
- 驱动层可替换

### 5. 配置驱动
- 基于YAML的配置管理
- 类型安全的配置模型
- 配置热更新支持

### 6. 多语言支持
- 基于Qt的国际化框架
- 支持中文/英文切换
- 动态语言加载

### 7. 权限控制
- 基于角色的访问控制
- 细粒度权限管理
- 操作审计追踪

### 8. 系统监控
- 分阶段系统自检
- 健康状态监控
- 错误恢复机制

## 部署和扩展

### 1. 部署架构
- 单机部署，支持多端口并行测试
- 配置文件驱动，易于环境切换
- 日志本地存储，支持远程传输

### 2. 扩展点
- **新测试步骤**: 通过注册系统添加
- **新MES厂商**: 通过适配器模式集成
- **新仪器类型**: 通过驱动层扩展
- **新通信协议**: 通过协议解析器添加

### 3. 集成接口
- **MES集成**: RESTful API或自定义协议
- **数据库集成**: 支持多种数据库后端
- **监控集成**: 支持外部监控系统
- **报表集成**: 支持多种报表格式

## 总结

该生产线测试工具采用了现代化的软件架构设计，具有以下优势：

1. **模块化设计**: 各模块职责清晰，便于维护和扩展
2. **松耦合架构**: 通过事件总线和依赖注入实现模块间松耦合
3. **异步编程**: 支持高并发和实时响应
4. **配置驱动**: 灵活的配置管理，支持多环境部署
5. **权限控制**: 完善的安全机制，支持多角色管理
6. **系统监控**: 全面的自检和监控机制
7. **扩展性强**: 插件化设计，易于功能扩展

该架构为智能硬件生产线测试提供了可靠、高效、可扩展的解决方案。
