# 变量使用说明 - engineer.test 步骤

## 🎯 功能说明

`engineer.test` 步骤现在支持**变量替换**功能，可以在命令中引用前面步骤的结果。

---

## ✨ 支持的变量格式

### 1. `${sn}` - 产品序列号

从 `scan.sn` 步骤扫描的序列号。

**示例**:
```yaml
- id: step_1
  name: ScanSN
  type: scan.sn

- id: step_2
  name: WritePN
  type: engineer.test
  params:
    command: 'pn=1,s100,${sn}%'  # ${sn} 会被替换为扫描的序列号
```

**实际执行**:
- 如果扫描的 SN 是 `ABC123456`
- 命令会变成: `pn=1,s100,ABC123456%`

---

### 2. `${port}` - 测试端口

当前测试端口名称（如 "PortA", "PortB"）。

**示例**:
```yaml
params:
  command: 'testinfo=1,machine,${port}%'
```

---

### 3. `${context.key}` - 上下文中的任意值

从测试上下文状态中获取任意键值。

**示例**:
```yaml
params:
  command: 'version=0,${context.firmware_version}%'
```

## 📝 完整示例

### 示例1: 写入 S100 PN（使用扫描的 SN）

```yaml
version: '1.0'
metadata:
  name: WriteS100PN
  product: Vita01

steps:
  # 步骤1: 扫描序列号
  - id: step_1
    name: ScanSN
    type: scan.sn
    timeout: 5000
    retries: 3

  # 步骤2: 进入工程模式
  - id: step_2
    name: EnterEngineerMode
    type: engineer.test
    params:
      command: 'enfac=1,1%'
      robot_ip: '192.168.125.2'
    timeout: 35000
    retries: 2

  # 步骤3: 写入S100的PN（使用扫描的SN）
  - id: step_3
    name: WriteS100PN
    type: engineer.test
    params:
      command: 'pn=1,s100,${sn}%'    # ${sn} 自动替换
      robot_ip: '192.168.125.2'
    timeout: 35000
    retries: 2
```

**执行过程**:
1. 用户扫描 SN: `VITA2024001`
2. step_3 执行时，`${sn}` 被替换为 `VITA2024001`
3. 实际执行命令: `pn=1,s100,VITA2024001%`

## 🔧 PN 命令说明

### 命令格式
```
pn=<op>,<device>,<value>%
```

### 参数说明

| 参数 | 说明 | 可选值 |
|------|------|--------|
| `op` | 操作类型 | `0`=读取, `1`=写入 |
| `device` | 设备类型 | `sn`, `s100`, `x5` |
| `value` | 序列号值 | 任意字符串（写入时必需） |

### 常用命令示例

```yaml
# 读取命令（查询）
command: 'pn=0,sn%'        # 读取主板SN
command: 'pn=0,s100%'      # 读取S100 PN
command: 'pn=0,x5%'        # 读取X5 PN

# 写入命令（使用变量）
command: 'pn=1,sn,${sn}%'     # 写入主板SN
command: 'pn=1,s100,${sn}%'   # 写入S100 PN
command: 'pn=1,x5,${sn}%'     # 写入X5 PN

# 写入命令（固定值）
command: 'pn=1,s100,VITA123%'  # 写入固定值
```

## 📊 变量替换过程

### 流程图

```
step_1: scan.sn
    ↓
[用户扫描] SN = "VITA2024001"
    ↓
[保存到上下文] ctx.set_sn("VITA2024001")
    ↓
step_3: engineer.test
    ↓
[读取参数] command = "pn=1,s100,${sn}%"
    ↓
[替换变量] ${sn} → "VITA2024001"
    ↓
[执行命令] "pn=1,s100,VITA2024001%"
    ↓
[发送到机器人] 192.168.125.2:3579
```

## ⚠️ 注意事项

### 1. 变量未定义

如果变量不存在，会保持原样并记录警告：

```yaml
command: 'pn=1,s100,${undefined}%'
# ${undefined} 不会被替换，命令保持原样
```

**日志输出**:
```
WARNING | 未找到变量: ${undefined}
```

---

### 2. SN 必须先扫描

确保在使用 `${sn}` 之前已经执行了 `scan.sn` 步骤：

**✅ 正确**:
```yaml
steps:
  - id: step_1
    type: scan.sn          # 先扫描
  - id: step_2
    type: engineer.test
    params:
      command: 'pn=1,s100,${sn}%'  # 再使用
```

**❌ 错误**:
```yaml
steps:
  - id: step_1
    type: engineer.test
    params:
      command: 'pn=1,s100,${sn}%'  # ${sn} 还未定义！
  - id: step_2
    type: scan.sn
```

---

### 3. 变量格式

- ✅ 正确: `${sn}`
- ❌ 错误: `$sn`, `{sn}`, `$(sn)`

## ✅ 总结

### 变量使用要点

1. **格式**: `${variable_name}`
2. **SN 变量**: `${sn}` - 从 scan.sn 步骤获取
3. **执行顺序**: 先扫描 SN，再使用 `${sn}`
4. **日志查看**: 会记录变量替换过程

### 命令示例

```yaml
# 写入 S100 PN（使用扫描的 SN）
command: 'pn=1,s100,${sn}%'

# 写入 X5 PN（使用扫描的 SN）
command: 'pn=1,x5,${sn}%'

# 写入主板 SN（使用扫描的 SN）
command: 'pn=1,sn,${sn}%'
```
