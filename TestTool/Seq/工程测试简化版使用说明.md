# 工程测试简化版使用说明

## 🎯 新增功能

为了解决连接问题和简化配置，新增了 `engineer.test` 步骤类型，它可以**一步完成**连接、测试和断开。

---

## ✨ engineer.test 步骤类型

### 功能特点

- ✅ **一步到位**: 自动管理连接、测试、断开全流程
- ✅ **简单配置**: 只需指定测试用例和机器人IP
- ✅ **兼容性强**: 使用已验证的 `test_engineer_client.py` 实现
- ✅ **稳定可靠**: 与 light_simple.yaml 相同的实现方式

### 支持的测试方式

#### 方式1: 使用命令字符串（推荐）

```yaml
- id: test_step
  name: 执行工程命令
  type: engineer.test
  params:
    command: 'enfac=1,1%'      # 完整的命令字符串
    robot_ip: '192.168.125.2'
    port: 3579
    timeout: 30
```

#### 方式2: 使用测试用例名

```yaml
- id: test_light
  name: 耳朵灯测试
  type: engineer.test
  params:
    test_case: 'light'          # 测试用例名
    robot_ip: '192.168.125.2'
    port: 3579
    timeout: 30
```

---

## 📝 配置参数说明

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `command` | 否 | - | 完整命令字符串（如 `enfac=1,1%`） |
| `test_case` | 否 | - | 测试用例名（如 `light`, `battery`） |
| `robot_ip` | 否 | `192.168.125.2` | 机器人IP地址 |
| `port` | 否 | `3579` | 机器人端口 |
| `timeout` | 否 | `30` | 超时时间（秒） |

**注意**: `command` 和 `test_case` 必须提供其中一个，优先使用 `command`。

## 🚀 三种YAML文件对比

### 文件A: Vita01-MT-1.0-Simple.yaml ⭐⭐⭐ 强烈推荐

**特点**: 使用 `engineer.test` 一步到位

```yaml
steps:
  - ScanSN                      # 扫描序列号
  - EnterEngineerMode           # 执行工程命令（自动连接+测试+断开）
    type: engineer.test
    params:
      command: 'enfac=1,1%'
      robot_ip: '192.168.125.2'
```

**优点**:
- ✅ 最简单（2步）
- ✅ 自动管理连接
- ✅ 稳定可靠
- ✅ **强烈推荐生产环境使用**

---

### 文件B: Vita01-MT-1.0-EngineerMode.yaml

**特点**: 使用 `connect_engineer` 自动进入工程模式

```yaml
steps:
  - ScanSN                      # 扫描序列号
  - ConnectEngineer             # 连接并自动进入工程模式
    type: connect_engineer
    params:
      host: '192.168.125.2'
```

**优点**:
- ✅ 保持连接（可继续执行其他命令）
- ✅ 适合需要多次操作的场景

**缺点**:
- ⚠️ 需要手动管理连接生命周期
- ⚠️ 可能遇到连接状态问题

---

### 文件C: Vita01-MT-1.0-EnfacCommand.yaml

**特点**: 使用 `connect_engineer` + `engineer.command`

```yaml
steps:
  - ScanSN                      # 扫描序列号
  - ConnectEngineer             # 连接工程服务
  - ExecuteEnfac                # 执行命令
    type: engineer.command
    params:
      command: 'enfac=1,1%'
```

**优点**:
- ✅ 最灵活（可执行多个命令）
- ✅ 显式控制每个步骤

**缺点**:
- ⚠️ 最复杂（3步）
- ⚠️ 需要手动管理连接

## 🎯 推荐使用方案

### 场景1: 简单的工程命令测试 ⭐⭐⭐

**推荐**: `Vita01-MT-1.0-Simple.yaml`

```yaml
- type: engineer.test
  params:
    command: 'enfac=1,1%'
    robot_ip: '192.168.125.2'
```

**为什么**: 一步到位，简单稳定

---

### 场景2: 需要执行多个命令

**推荐**: `Vita01-MT-1.0-Simple.yaml` + 多个 engineer.test 步骤

```yaml
steps:
  - id: step_1
    name: ScanSN
    type: scan.sn

  - id: step_2
    name: EnterEngineer
    type: engineer.test
    params:
      command: 'enfac=1,1%'
      robot_ip: '192.168.125.2'

  - id: step_3
    name: QueryVersion
    type: engineer.test
    params:
      command: 'version=0%'
      robot_ip: '192.168.125.2'

  - id: step_4
    name: QueryBattery
    type: engineer.test
    params:
      command: 'battery=0%'
      robot_ip: '192.168.125.2'

  - id: step_5
    name: ExitEngineer
    type: engineer.test
    params:
      command: 'enfac=1,0%'
      robot_ip: '192.168.125.2'
```

**为什么**: 每个命令独立连接，互不影响

---

### 场景3: 需要保持连接执行多个命令

**推荐**: `Vita01-MT-1.0-EnfacCommand.yaml`

```yaml
steps:
  - id: step_1
    type: scan.sn

  - id: step_2
    type: connect_engineer      # 连接一次
    params:
      host: '192.168.125.2'

  - id: step_3
    type: engineer.command      # 命令1
    params:
      command: 'enfac=1,1%'

  - id: step_4
    type: engineer.command      # 命令2
    params:
      command: 'version=0%'

  - id: step_5
    type: engineer.command      # 命令3
    params:
      command: 'battery=0%'

  - id: step_6
    type: disconnect_engineer   # 断开连接
```

**为什么**: 只连接一次，执行效率高

## ⚠️ 故障排查

### 问题1: 找不到 test_engineer_client.py

**错误**: `找不到工程测试客户端`

**解决**:
```bash
# 确认 client 目录结构正确
client/
  vita_engineer_client/
    test_engineer_client.py  ← 必须存在
```

---

### 问题2: 导入失败

**错误**: `无法导入工程测试客户端`

**解决**:
```bash
# 安装依赖
cd client
pip install -r requirements.txt
```

---

### 问题3: 连接超时

**错误**: `工程测试超时`

**解决**:
1. 检查机器人IP: `ping 192.168.125.2`
2. 检查端口: `telnet 192.168.125.2 3579`
3. 增加超时时间:

```yaml
params:
  timeout: 60  # 增加到60秒
```
