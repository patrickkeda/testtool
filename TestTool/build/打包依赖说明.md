# TestTool 打包依赖说明

## 概述

本文档说明 `vita-servo-id2.yaml` 测试序列所需的依赖环境，以及如何通过 `build_exe.bat` 脚本将所有依赖打包到 `dist` 目录，确保打包后的程序能在其他电脑上运行。

## vita-servo-id2.yaml 依赖分析

### 1. 测试步骤类型

该 YAML 文件使用了以下步骤类型：

- `scan.sn` - 扫描序列号
- `plc.modbus.connect` - PLC Modbus RTU 串口连接（115200 8O1）
- `plc.modbus.write_register` - PLC 写寄存器/线圈
- `plc.modbus.read_register` - PLC 读寄存器/线圈
- `can.connect` - CAN 总线设备连接
- `can.send_frame` - CAN 帧发送
- `utility.delay` - 延迟等待
- `plc.modbus.disconnect` - PLC 断开连接
- `can.disconnect` - CAN 断开连接

### 2. Python 库依赖

#### 核心通信库
- **pymodbus** (>=3.5.0) - Modbus RTU 串口通信
  - `pymodbus.client.serial.ModbusSerialClient` - 串口 Modbus 客户端
- **pyserial** (>=3.5) - 底层串口通信支持
  - pymodbus 依赖此库进行串口操作

#### GUI 框架
- **PySide6** (>=6.5.0) - Qt GUI 框架

#### 配置和数据处理
- **pyyaml** (>=6.0) - YAML 配置文件解析
- **pydantic** (>=2.0.0) - 数据验证和模型

#### CAN 总线支持
- **ctypes** - Python 标准库，用于加载 DLL
- **ECanVci64.dll** / **CHUSBDLL64.dll** - CAN 设备驱动 DLL（Windows）

#### 其他依赖
- **asyncio** - 异步事件循环（Python 标准库）
- **threading** - 多线程支持（Python 标准库）
- **pathlib** - 路径处理（Python 标准库）

### 3. 文件依赖

#### 配置文件
- `Config/config.yaml` - 主配置文件
- `Seq/vita-servo-id2.yaml` - 测试序列文件

#### CAN 相关文件
- `test/canapp/can_sender.py` - CAN 协议发送器模块
- `test/canapp/ECanVci64.dll` - CAN 设备 DLL（64位）
- `test/canapp/CHUSBDLL64.dll` - CAN 设备 DLL（64位，备用）

### can_sender.py DLL 加载逻辑更新

更新了 `_load_dll()` 方法，支持 PyInstaller 打包环境：

1. **PyInstaller 临时目录** (`sys._MEIPASS`) - 打包后 DLL 的临时解压位置
2. **标准打包位置** (`_internal/test/canapp/`) - exe 所在目录的标准位置
3. **当前工作目录** - 开发环境支持
4. **模块所在目录** - 直接运行脚本时支持

## 打包步骤

### 1. 确保依赖已安装

运行以下命令安装所有依赖：

```bash
pip install -r requirements.txt
pip install pyinstaller
```

### 2. 运行打包脚本

```bash
cd TestTool
build\build_exe.bat
```

### 3. 验证打包结果

打包完成后，检查 `dist\TestTool\` 目录结构：

```
dist\TestTool\
├── TestTool.exe              # 主程序
└── _internal\                # 依赖和资源文件
    ├── Config\               # 配置文件目录
    ├── Seq\                  # 测试序列目录（包含 vita-servo-id2.yaml）
    ├── test\canapp\          # CAN 相关模块
    │   ├── can_sender.py     # CAN 发送器模块
    │   ├── ECanVci64.dll     # CAN 设备 DLL
    │   └── CHUSBDLL64.dll    # CAN 设备 DLL（备用）
    └── [其他依赖文件]
```

## 部署到其他电脑

### 1. 复制整个目录

将 `dist\TestTool\` 整个文件夹复制到目标电脑。

### 2. 系统要求

- **操作系统**: Windows 7/10/11 (64位)
- **Python**: 不需要安装（已打包在 exe 中）
- **其他依赖**: 不需要安装（已打包）

### 3. CAN 设备驱动

如果使用 CAN 设备，需要确保：
- CAN 设备的 USB 驱动已安装（由设备厂商提供）
- DLL 文件（ECanVci64.dll）与设备驱动兼容

### 4. 串口权限

确保运行程序的用户有权限访问 COM 口（通常需要管理员权限或配置串口权限）。

### 5. 配置文件修改

首次部署时，可能需要修改 `_internal\Config\config.yaml` 中的配置：
- 串口号（如 COM3）
- 其他设备参数

## 常见问题

### Q1: DLL 加载失败

**原因**: DLL 文件未正确打包或路径不正确

**解决**:
1. 检查 `dist\TestTool\_internal\test\canapp\` 目录是否存在 DLL 文件
2. 确认 DLL 文件与 Python 版本（32位/64位）匹配
3. 检查 Windows 系统是否缺少 Visual C++ 运行库

### Q2: Modbus 连接失败

**原因**: 串口参数不匹配或串口被占用

**解决**:
1. 确认 YAML 文件中的串口参数（115200 8O1）
2. 检查串口是否被其他程序占用
3. 确认 PLC 设备在线且 unit_id 正确

### Q3: CAN 设备无法连接

**原因**: CAN 设备驱动未安装或 DLL 不兼容

**解决**:
1. 安装 CAN 设备的 USB 驱动
2. 确认 DLL 版本与设备驱动兼容
3. 检查设备管理器中 CAN 设备是否正常识别
