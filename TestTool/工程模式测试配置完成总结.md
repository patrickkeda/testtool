class EngineerCommandStep(BaseStep):

## 📝 YAML配置示例

### 完整的工程模式测试流程

```yaml
version: '1.0'
metadata:
  name: 完整工程模式测试
  product: Vita01
  station: MT

steps:
  # 步骤1: 扫描SN
  - id: step_1
    name: ScanSN
    type: scan.sn
    timeout: 5000
    retries: 3

  # 步骤2: 连接工程服务
  - id: step_2
    name: ConnectEngineer
    type: connect_engineer
    params:
      host: '192.168.125.2'
      port: 3579
      timeout_ms: 10000
    timeout: 15000
    retries: 3

  # 步骤3: 执行enfac命令（进入工程模式）
  - id: step_3
    name: EnterEngineerMode
    type: engineer.command
    params:
      command: 'enfac=1,1%'
      timeout_ms: 5000
      expect_success: true
    timeout: 10000
    retries: 2

  # 步骤4: 查询版本（可选）
  - id: step_4
    name: QueryVersion
    type: engineer.command
    params:
      command: 'version=0%'
      timeout_ms: 3000

  # 步骤5: 查询电池（可选）
  - id: step_5
    name: QueryBattery
    type: engineer.command
    params:
      command: 'battery=0%'
      timeout_ms: 3000

  # 步骤6: 退出工程模式（可选）
  - id: step_6
    name: ExitEngineerMode
    type: engineer.command
    params:
      command: 'enfac=1,0%'
      timeout_ms: 5000

  # 步骤7: 断开连接
  - id: step_7
    name: DisconnectEngineer
    type: disconnect_engineer
    timeout: 5000
```

## ⚠️ 重要提示

### 1. 命令格式
- ✅ 正确: `enfac=1,1%` （必须以%结尾）
- ❌ 错误: `enfac=1,1` （缺少%）
- ❌ 错误: `enfac 1 1%` （缺少=和逗号）

### 2. 执行顺序
```
必须先连接 → 才能执行命令
connect_engineer → engineer.command
```

### 3. IP地址配置
- 当前配置: `192.168.125.2:3579`
- 可在YAML中修改
- 也可在 `Config/config.yaml` 中统一配置

### 4. 超时设置
| 命令类型 | 建议超时 |
|----------|----------|
| 查询命令 | 3000-5000ms |
| 控制命令 | 5000-10000ms |
| 相机拍照 | 10000ms |
| 雷达扫描 | 15000ms |
